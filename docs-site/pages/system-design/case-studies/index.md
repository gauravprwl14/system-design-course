## 🎯 How to Use These Case Studies

Each case study follows this structure:

1. **Problem Statement** - What are we building?
2. **Requirements** - Functional and non-functional
3. **Capacity Estimation** - Scale calculations
4. **System APIs** - Interface design
5. **Database Schema** - Data modeling
6. **Architecture Diagram** - High-level design
7. **Deep Dives** - Component details
8. **Trade-offs** - Decisions and alternatives
9. **Scaling** - How to handle growth
10. **Implementation** - Pseudocode examples

## 📊 Difficulty Guide

- 🟡 **Intermediate**: Common interview questions, 45-60 min interviews
- 🔴 **Advanced**: Complex systems, 60+ min interviews or real projects
