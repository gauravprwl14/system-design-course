# 14-algorithms/hands-on/ — Layer 2 Router

Runnable implementations of core data structures and algorithms — each POC is self-contained with explanation, code, and complexity analysis.

## Files in This Section

| File | Description |
|------|-------------|
| overview | POC index and prerequisites |
| bloom-filter-poc | Bloom filter from scratch: bit array, multiple hash functions, false positive rate tuning |
| lru-cache-poc | LRU Cache implementation using doubly-linked list + hash map in O(1) get/put |
| consistent-hashing-poc | Consistent hashing ring with virtual nodes: add/remove nodes, key lookup |
| trie-autocomplete-poc | Trie (prefix tree) with autocomplete: insert, search, prefix query, top-K suggestions |
| external-sort-poc | External sort (k-way merge): sort a file larger than RAM using chunk-and-merge |
| backtracking-poc | Backtracking template: N-queens, permutations, Sudoku solver |
| graph-shortest-path-poc | Graph shortest path: BFS for unweighted, Dijkstra for weighted, with adjacency list |
| string-search-poc | String search algorithms: KMP and Rabin-Karp implemented and benchmarked |
| segment-tree-poc | Segment tree: range sum query and point update with lazy propagation |
