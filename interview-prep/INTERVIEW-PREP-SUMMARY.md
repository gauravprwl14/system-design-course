## ğŸ’¡ Interview Tips

### Answering System Design Questions (45 min)
1. **Clarify requirements** (5 min)
   - Functional vs non-functional
   - Scale estimates
   - Out of scope

2. **High-level design** (10 min)
   - Draw architecture diagram
   - Identify main components
   - Explain data flow

3. **Deep dive** (20 min)
   - Database schema
   - API design
   - Scaling strategy
   - Handle edge cases

4. **Trade-offs** (5 min)
   - Discuss alternatives
   - Cost vs performance
   - Consistency vs availability

5. **Wrap up** (5 min)
   - Monitoring approach
   - Security considerations
   - Future improvements

### Answering Technical Questions (5 min)
1. **Quick answer** (30 sec)
   - Core concept in 2-3 sentences
   - Show you know the answer

2. **Expand with details** (2 min)
   - How it works
   - When to use
   - Key characteristics

3. **Provide example** (1 min)
   - Real-world usage
   - Code snippet if relevant
   - Company examples

4. **Discuss trade-offs** (30 sec)
   - Pros and cons
   - Alternatives
   - Best practices

5. **Invite follow-up** (30 sec)
   - "Happy to dive deeper into..."
   - Show breadth of knowledge
